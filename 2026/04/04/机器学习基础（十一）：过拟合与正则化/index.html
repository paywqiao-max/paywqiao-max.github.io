机器学习基础（十一）：过拟合与正则化

一、引言

在第十篇学习 CNN 时，我们已经用到了 Dropout 和 BatchNorm。你有没有好奇过：

为什么要"随机丢弃神经元"？加一个"批归一化层"到底在做什么？

这些技巧，都是为了解决深度学习中最核心的挑战之一——过拟合（Overfitting）。

本篇从「过拟合的本质」出发，系统讲解各类正则化方法的原理与实践，帮你真正理解为什么要这样做，而不只是照着用。

二、过拟合与欠拟合

1. 什么是过拟合？

过拟合：模型在训练集上表现很好，在测试集（未见过的数据）上表现很差。

类比：学生死记硬背了所有例题答案，但一换题就不会了。

欠拟合：模型连训练集都没学好，训练集和测试集表现都差。

类比：学生根本没认真学，连例题都做不对。

2. 过拟合的根本原因

  • 模型太复杂：参数太多，足以"背下"整个训练集
  • 训练数据太少：无法覆盖真实分布的多样性
  • 训练时间太长：模型开始记住训练集的噪声

3. 偏差与方差（Bias-Variance Tradeoff）

这是理解过拟合/欠拟合的核心框架：

$$\text{泛化误差} = \text{偏差}^2 + \text{方差} + \text{噪声}$$

偏差（Bias） 方差（Variance）
含义 模型对真实规律的拟合程度 模型对数据扰动的敏感程度
欠拟合 高 低
过拟合 低 高
目标 同时降低 同时降低

直观理解：

  • 高偏差 = 瞄准靶心的方向就错了（系统性偏差）
  • 高方差 = 每次射击落点差异很大（不稳定）
  • 理想：落点集中且靠近靶心

三、正则化方法全景

正则化（Regularization）是减少过拟合、提升泛化能力的各类技术的统称。主要分为以下几类：

四、L1 和 L2 正则化（权重衰减）

1. 核心思想

在损失函数中添加惩罚项，限制模型参数的大小，防止模型过度复杂。

$$\mathcal{L}{total} = \mathcal{L}{task} + \lambda \cdot \Omega(W)$$

其中 $\lambda$ 是正则化强度，$\Omega(W)$ 是惩罚项。

2. L2 正则化（Ridge / 权重衰减）

$$\Omega(W) = |W|2^2 = \sum{i} w_i^2$$

效果：鼓励权重接近 0，但不会变成精确的 0，整体权重趋于"小而均匀"。

梯度更新（等价于权重衰减）：

$$w \leftarrow w - \eta \frac{\partial \mathcal{L}}{\partial w} - \eta \lambda w$$

每次更新时，权重额外乘以 $(1 - \eta\lambda)$，被轻微"拉向零"。

3. L1 正则化（Lasso）

$$\Omega(W) = |W|1 = \sum{i} |w_i|$$

效果：会产生稀疏解，很多权重会变成精确的 0。

类比：L2 是"要求所有人贡献，但限制每人贡献量"；L1 是"直接淘汰贡献小的人"。

五、Dropout

1. 核心思想

训练时，随机将部分神经元的输出置为 0，相当于每次训练都用了一个不同的"瘦网络"。

类比：一个团队里，每次开会随机让一些人缺席。这样每个人都要独当一面，不能依赖别人，整个团队的鲁棒性更强。

2. 工作原理

  • 训练阶段：每个神经元以概率 $p$ 被"丢弃"（输出设为 0），保留概率为 $(1-p)$
  • 测试阶段：所有神经元都参与，但输出乘以保留率 $(1-p)$，保持期望一致

现代实现通常用反向 Dropout（Inverted Dropout）：

这样测试时不需要额外的缩放操作。

3. 为什么有效？

Dropout 从多个角度可以解释其有效性：

  1. 集成学习视角：每次训练相当于训练了一个不同的子网络，最终相当于对指数级数量的网络取平均
  2. 减少神经元共适应：防止某些神经元过度依赖特定的其他神经元，迫使每个神经元学习更鲁棒的特征
  3. 增加噪声：相当于给激活值加入随机噪声，起到正则化效果

4. PyTorch 实现

⚠️ 重要：一定要在测试/评估时调用 model.eval()，否则 Dropout 仍会生效，导致结果不稳定！

5. Dropout 使用建议

位置 常用丢弃率
全连接层之后 0.5
卷积层之后 0.1 ~ 0.3（通常效果有限）
输入层 0.1 ~ 0.2
  • Transformer 等现代架构中，Dropout 通常设为 0.1
  • 数据越少，dropout 率可以适当调高
  • CNN 中的 Dropout 效果不如全连接网络显著，可用 DropBlock 替代

六、批归一化（Batch Normalization）

1. 问题背景：Internal Covariate Shift

深度网络训练时，每层的输入分布会随着前面层的参数更新而不断变化，这称为内部协变量偏移（Internal Covariate Shift）。

影响：

  • 需要使用较小的学习率，收敛慢
  • 需要仔细初始化参数
  • 训练不稳定，容易出现梯度消失/爆炸

2. BatchNorm 的核心思想

在每个 mini-batch 上，对每个特征维度进行归一化，然后用可学习的参数缩放和平移：

$$\hat{x}_i = \frac{x_i - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}}$$

$$y_i = \gamma \hat{x}_i + \beta$$

其中：

  • $\mu_B$：当前 batch 的均值
  • $\sigma_B^2$：当前 batch 的方差
  • $\epsilon$：防止除零的小常数（如 $10^{-5}$）
  • $\gamma, \beta$：可学习的缩放和平移参数（模型自己学习最优的分布）

3. 为什么有效？

  1. 稳定分布：让每层输入保持相似的分布，加速收敛
  2. 正则化效果：由于使用 batch 统计量，引入了随机噪声，有轻微正则化作用
  3. 允许更大学习率：分布稳定后，可以用更大的学习率训练
  4. 减少对初始化的依赖：即使初始化不完美，BatchNorm 也能稳定训练

4. 训练 vs 推理的差异

阶段 均值/方差来源 特点
训练 当前 mini-batch 每次不同，有随机性
推理 训练时积累的全局统计量（移动平均） 确定性输出

这就是为什么 model.eval() 很重要——它切换了 BatchNorm 的统计量来源！

5. PyTorch 实现

💡 顺序建议：Conv → BN → ReLU（原始论文建议），也有人用 Conv → ReLU → BN，实践中差别不大。

6. 各类归一化对比

不同场景下，BatchNorm 并非总是最佳选择：

方法 归一化维度 适用场景 PyTorch

七、数据增强（Data Augmentation）

1. 为什么有效？

增加训练数据的多样性，让模型见到更多变化，从根本上减少过拟合。

本质：数据增强 = 利用领域知识构造等价变换

九、标签平滑（Label Smoothing）

1. 问题：过度自信

交叉熵损失函数驱使模型对正确类别输出尽可能高的概率（趋近 1），对其他类别输出 0。这会让模型过度自信，泛化能力差。

2. 标签平滑的做法

将硬标签（0/1）软化为软标签：

$$y_{smooth} = (1 - \epsilon) \cdot y_{hard} + \frac{\epsilon}{K}$$

其中 $\epsilon$ 是平滑系数（通常 0.1），$K$ 是类别数。

例子：3 类分类，$\epsilon = 0.1$

  • 原始标签：[1, 0, 0]
  • 平滑后：[0.933, 0.033, 0.033]

十二、总结

本篇系统讲解了过拟合的本质与各类正则化手段：

  1. 偏差-方差权衡是理解过拟合/欠拟合的核心框架
  2. L1/L2 正则化从损失函数层面约束参数大小
  3. Dropout通过随机丢弃实现集成效果，切记 model.eval()
  4. BatchNorm稳定训练过程，是现代深度网络的标配
  5. 数据增强是最直接有效的正则化手段
  6. 早停 + 标签平滑是工程实践中的实用技巧

正则化没有银弹。最好的策略是：先把模型做到过拟合（证明容量足够），再逐步加正则化拉回来。
